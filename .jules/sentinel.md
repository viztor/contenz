## 2025-04-24 - [Arbitrary Code Execution in MDX Adapter]
**Vulnerability:** The MDX adapter (`packages/adapter-mdx/src/index.ts`) was using `new Function` to evaluate object literals representing metadata exported from MDX files. This could allow for Arbitrary Code Execution (ACE) if an attacker provides a maliciously crafted MDX file containing executable JavaScript code in the metadata export.
**Learning:** The vulnerability existed because `new Function` evaluates strings as JavaScript code within the global scope, meaning any JavaScript provided in the string would be executed with the privileges of the Node.js process parsing the file. This violates the principle of safe evaluation of untrusted input.
**Prevention:** Replaced `new Function` with `runInNewContext` from the `node:vm` module. This provides an isolated context (`Object.create(null)`) and an execution timeout to safely evaluate the object literal, preventing it from accessing the global environment or running indefinitely. Always avoid `new Function` or `eval` for dynamic object evaluation.
